## 1. PRINCIPES FONDAMENTAUX

### 1.1 — 10 règles immuables

1. **Jamais hardcoder une langue** — tout passe par `t("key")`
2. **FR par défaut**, EN fallback — en cas de clé manquante
3. **Détection auto** de la langue user au 1er lancement
4. **Persistance** du choix de langue dans `users.locale`
5. **Même qualité** dans les 6 langues (pas de "FR first, autres moches")
6. **Traductions validées par natifs** (pas de Google Translate aveugle)
7. **Adaptation culturelle** (Ramadan, Shabbat, formules de politesse)
8. **RTL correct en arabe** (layout, icônes flippées, scrolling)
9. **Pluriels ICU** pour toutes les quantités variables
10. **Fallback gracieux** — ne jamais afficher une clé brute (`user.profile.title`)

### 1.2 — Pourquoi c'est stratégique

**Marché** : ~500K artisans en France (voir `DISTRIBUTION.md`)
- ~85% franco-phones
- **~15% langue maternelle non-française** : turc, arabe maghrébin, portugais, espagnol = **~75 000 artisans**
- **0 concurrent** ne les adresse → marché invisible

**Plus** : expansion Europe post-launch facile (langues déjà dans le produit).

### 1.3 — Portée i18n

**Ce qui est traduit** :
- Interface mobile (tous les écrans)
- Emails transactionnels
- Templates WhatsApp (84 total)
- Messages d'erreur user-facing
- Mentions obligatoires devis/factures
- PDF devis/factures (F135 multi-pays)
- Landings marketing (/tr /ar /pt /es)
- Capsules formation (voix-off)
- Notifications push

**Ce qui n'est PAS traduit** :
- Code backend (anglais, standard dev)
- Logs internes (anglais)
- Documentation technique (français)
- Prompts LLM système (structure FR, Claude comprend tout)
- Noms propres (produits BTP, fournisseurs)

### 3.3 — Config i18next

**Fichier** : `mobile/i18n/index.ts`

```typescript
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import * as Localization from 'expo-localization';
import AsyncStorage from '@react-native-async-storage/async-storage';

import fr from './fr.json';
import en from './en.json';
import tr from './tr.json';
import pt from './pt.json';
import ar from './ar.json';
import es from './es.json';

const SUPPORTED_LANGUAGES = ['fr', 'en', 'tr', 'pt', 'ar', 'es'] as const;
type SupportedLanguage = typeof SUPPORTED_LANGUAGES[number];

const languageDetector = {
  type: 'languageDetector' as const,
  async: true,
  detect: async (callback: (lang: string) => void) => {
    // 1. AsyncStorage (choix user précédent)
    const stored = await AsyncStorage.getItem('user_locale');
    if (stored && SUPPORTED_LANGUAGES.includes(stored as SupportedLanguage)) {
      return callback(stored);
    }
    
    // 2. Device locale
    const deviceLocale = Localization.locale.split('-')[0];
    if (SUPPORTED_LANGUAGES.includes(deviceLocale as SupportedLanguage)) {
      return callback(deviceLocale);
    }
    
    // 3. Fallback FR
    callback('fr');
  },
  init: () => {},
  cacheUserLanguage: async (lang: string) => {
    await AsyncStorage.setItem('user_locale', lang);
  },
};

i18n
  .use(languageDetector)
  .use(initReactI18next)
  .init({
    fallbackLng: 'fr',
    supportedLngs: SUPPORTED_LANGUAGES as unknown as string[],
    resources: { fr, en, tr, pt, ar, es },
    interpolation: {
      escapeValue: false,     // React échappe déjà
    },
    react: {
      useSuspense: false,
    },
    compatibilityJSON: 'v4',   // ICU plural rules v4
  });

export default i18n;
```

import { useTranslation } from 'react-i18next';

export const useI18n = () => {
  const { t, i18n } = useTranslation();
  
  const changeLanguage = async (lang: string) => {
    await i18n.changeLanguage(lang);
    
    // Appliquer RTL si nécessaire

      I18nManager.forceRTL(isRTL);

    await apiClient.patch('/v1/settings', { locale: `${lang}-${lang.toUpperCase()}` });
  };
  
  return { t, changeLanguage, currentLanguage: i18n.language };
};

## 4. ARCHITECTURE BACKEND (i18n_service)

### 4.1 — Quand traduire côté backend

Le backend traduit :

- Messages d'erreur API (`StructorAIError.message_fr` + `message_en`)

### 4.2 — Service

**Fichier** : `backend/app/services/i18n_service.py`

```python
from pathlib import Path
import json
from typing import Any

class I18nService:
    def __init__(self, translations_dir: Path):
        self.translations = {}
        self.supported_languages = ['fr', 'en', 'tr', 'pt', 'ar', 'es']
        self.fallback = 'fr'
        self._load_translations(translations_dir)
    
    def _load_translations(self, base_dir: Path):
        for lang in self.supported_languages:
            file_path = base_dir / f"{lang}.json"
            with open(file_path, encoding='utf-8') as f:
                self.translations[lang] = json.load(f)
    
    def t(self, key: str, locale: str = 'fr', **vars: Any) -> str:
        """
        Récupère une traduction.
        
        Exemples:
            t('email.welcome.subject', locale='fr', name='Jean')
            t('error.devis_not_found', locale='en')
        """
        lang = self._resolve_lang(locale)
        
        # Navigation via notation pointée (ex: "email.welcome.subject")
        value = self._get_nested(self.translations[lang], key)
        
        # Fallback si manquant
        if value is None and lang != self.fallback:
            value = self._get_nested(self.translations[self.fallback], key)
        
        if value is None:
            # Clé absente partout — log warning + retourne la clé brute
            logger.warning(f"Missing translation key: {key}")
            return key
        
        # Interpolation variables
        if vars and isinstance(value, str):
            value = value.format(**vars)
        
        return value
    
    def _resolve_lang(self, locale: str) -> str:
        """fr-FR → fr, en-GB → en, etc."""
        lang = locale.split('-')[0].split('_')[0].lower()
        return lang if lang in self.supported_languages else self.fallback
    
    def _get_nested(self, d: dict, key: str) -> Any:
        parts = key.split('.')
        current = d
        for p in parts:
            if isinstance(current, dict) and p in current:
                current = current[p]
            else:
                return None
        return current
    
    def pluralize(self, key: str, count: int, locale: str = 'fr', **vars) -> str:
        """
        Pluriel avec règles ICU.
        
        Ex: 
            key 'devis.count': {
                "one": "1 devis",
                "other": "{count} devis"
            }
        """
        rule = self._plural_rule(locale, count)
        return self.t(f"{key}.{rule}", locale=locale, count=count, **vars)
    
    def _plural_rule(self, locale: str, count: int) -> str:
        """Retourne 'one', 'other', 'zero', 'two', 'few', 'many'."""
        # Implémentation simplifiée (utiliser babel ou icu pour prod)
        lang = self._resolve_lang(locale)
        if lang == 'ar':
            # Arabe a 6 formes
            if count == 0: return 'zero'
            if count == 1: return 'one'
            if count == 2: return 'two'
            if 3 <= (count % 100) <= 10: return 'few'
            if 11 <= (count % 100) <= 99: return 'many'
            return 'other'
        elif lang in ('fr', 'pt'):
            return 'one' if count <= 1 else 'other'
        else:  # en, tr, es
            return 'one' if count == 1 else 'other'
```

## 6. PLURIELS ICU

### 6.1 — Règles par langue

| Langue | Formes de pluriel |
|---|---|
| **Français** | `one` (0, 1), `other` (2+) |
| **Anglais** | `one` (1), `other` (0, 2+) |
| **Turc** | `one` (1), `other` (0, 2+) |
| **Portugais** | `one` (0, 1), `other` (2+) |
| **Arabe** | `zero`, `one`, `two`, `few` (3-10), `many` (11-99), `other` (100+) |
| **Espagnol** | `one` (1), `other` (0, 2+) |

### 6.2 — Implémentation i18next (mobile)

i18next v4 gère ICU plurals nativement avec `compatibilityJSON: 'v4'`.

```typescript
// Utilisation

// FR : "3 devis en attente"
// AR : "3 عروض أسعار معلقة"
```

### 6.3 — Backend Python

Utiliser `babel` pour pluralization :

```python
from babel.plural import PluralRule

def get_plural_form(locale: str, count: int) -> str:
    rule = PluralRule.parse_rule_from_cldr(locale)
    return rule(count)  # 'zero', 'one', 'two', 'few', 'many', 'other'
```

## 8. RTL ARABE

### 8.1 — Défis

L'arabe s'écrit **de droite à gauche** (right-to-left). Impact complet sur le layout :
- Menu burger passe à droite
- Flèches retour pointent à droite
- Texte aligné à droite
- Scrolling listes inversé
- Input cursors
- Animations directionnelles

### 8.2 — React Native

```typescript
import { I18nManager } from 'react-native';

// Forcer RTL
I18nManager.forceRTL(true);

// Nécessite restart de l'app pour prendre effet
// On peut le détecter et afficher un modal "Redémarrer l'app"
```

### 8.3 — Styling

```typescript
// Utiliser writingDirection
const styles = StyleSheet.create({
  text: {
    writingDirection: 'rtl',
    textAlign: 'right',  // Auto en RTL mode
  },
});

// Ou utiliser logical properties
// (paddingStart/paddingEnd au lieu de paddingLeft/paddingRight)
```

### 8.4 — Icônes

Certaines icônes doivent être flippées horizontalement en RTL :
- Flèches retour (←)
- Chevrons (›)
- Flèches "envoyer"
- Indicateurs de direction

D'autres NE doivent PAS être flippées :
- Logos
- Icônes de check (✓)
- Icônes sans direction (home, settings)

### 8.5 — Pattern pour icônes directionnelles

```typescript
import { I18nManager } from 'react-native';

<ArrowRight style={{ transform: [{ scaleX: I18nManager.isRTL ? -1 : 1 }] }} />
```

### 8.6 — PDF RTL

ReportLab supporte RTL avec :
- Font arabe (ex: `Noto Sans Arabic`)
- Direction paragraphe `rtl`
- Alignement droit

```python
from reportlab.platypus import Paragraph
from reportlab.lib.styles import ParagraphStyle
from reportlab.lib.enums import TA_RIGHT

style_ar = ParagraphStyle(
    name='Arabic',
    fontName='NotoSansArabic',
    fontSize=12,
    alignment=TA_RIGHT,
    textAnchor='end',
)

p = Paragraph(arabic_text, style_ar)
```

### 8.7 — Attention : mélange RTL + LTR

Quand du texte arabe contient du français (ex: montant "123 €") ou vice-versa, utiliser marqueurs Unicode :
- LRM (Left-to-Right Mark) : `\u200E`
- RLM (Right-to-Left Mark) : `\u200F`

## 9. DÉTECTION AUTOMATIQUE DE LANGUE

### 9.1 — Ordre de priorité

1. **User preference** (settings → `users.locale`)
2. **AsyncStorage** (ancien choix mobile)
3. **Device locale** (OS du téléphone)
4. **Whisper détection** (premier message vocal)
5. **Fallback français**

## 21. SCRIPTS ET OUTILS

### 21.1 — Validation CI

**Fichier** : `scripts/validate_i18n.py`

```python
#!/usr/bin/env python3
"""Valide la cohérence des fichiers i18n."""
import json
from pathlib import Path

LANGUAGES = ['fr', 'en', 'tr', 'pt', 'ar', 'es']
BASE_LANG = 'fr'

def flatten_dict(d, parent=''):
    keys = []
    for k, v in d.items():
        full_key = f"{parent}.{k}" if parent else k
        if isinstance(v, dict):
            keys.extend(flatten_dict(v, full_key))
        else:
            keys.append(full_key)
    return keys

def main():
    i18n_dir = Path('mobile/i18n')
    files = {lang: json.load(open(i18n_dir / f"{lang}.json")) for lang in LANGUAGES}
    
    base_keys = set(flatten_dict(files[BASE_LANG]))
    
    errors = []
    for lang in LANGUAGES:
        if lang == BASE_LANG:
            continue
        keys = set(flatten_dict(files[lang]))
        
        missing = base_keys - keys
        orphan = keys - base_keys
        
        for k in missing:
            errors.append(f"[{lang}] Missing key: {k}")
        for k in orphan:
            errors.append(f"[{lang}] Orphan key: {k}")
    
    if errors:
        print("\n".join(errors))
        exit(1)
    print(f"✅ All 6 languages have {len(base_keys)} consistent keys.")

if __name__ == '__main__':
    main()
```

Utilisé dans `.github/workflows/mobile-ci.yml` :

```yaml
- name: Validate i18n
  run: python scripts/validate_i18n.py
```

## 24. ANTI-PATTERNS

### 24.1 — À NE JAMAIS FAIRE

1. ❌ Hardcoder du texte en dur : `<Text>Bienvenue</Text>`
2. ❌ Concaténer des strings : `t('welcome') + ", " + name`
3. ❌ Utiliser Google Translate sans revue humaine
4. ❌ Oublier `return key` en fallback (afficher clé brute)
5. ❌ Utiliser des emojis comme identifiants (🇫🇷 pour "fr")
6. ❌ Hardcoder des dates (`"15 mars 2026"`)
7. ❌ Hardcoder des formats nombres (`"1 234,56 €"`)
8. ❌ Ignorer RTL en arabe (texte aligné gauche brutalement)
9. ❌ Utiliser la même voix vocale pour toutes les langues sans test
10. ❌ Oublier la langue du client pour PDF (utiliser celle de l'artisan par erreur)

### 24.2 — À FAIRE

1. ✅ Toujours `t('key', { vars })`
2. ✅ Toujours variables d'interpolation
3. ✅ Valider avec natifs avant publication
4. ✅ Fallback chaîné (demandé → FR → EN → clé brute + warning)
5. ✅ Codes BCP 47 standards
6. ✅ Utiliser `date-fns` / `Intl`
7. ✅ Utiliser `Intl.NumberFormat` / `Intl.DateTimeFormat`
8. ✅ Tester RTL sur device réel
9. ✅ Tester voice TTS sur phrases BTP typiques
10. ✅ Langue PDF = langue client

## 25. TROUBLESHOOTING

### 25.1 — Problèmes fréquents

| Symptôme | Cause | Solution |
|---|---|---|
| Clé brute affichée (`auth.login_btn`) | Clé manquante dans JSON | Ajouter la clé + traduire |
| Traduction en anglais alors que FR attendu | Détection langue wrong | Vérifier `users.locale` |
| RTL pas appliqué | `I18nManager.forceRTL` pas reload | Restart app requis |
| Pluriel incorrect en arabe | Règle ICU mal appliquée | Vérifier 6 formes AR |
| Date en format US alors que FR | `Intl` pas locale-aware | Passer `locale` explicite |
| PDF ar vide / caractères | Font arabe pas chargée | Embedder NotoSansArabic.ttf |
| Templates WhatsApp en FR pour user TR | Template pas approuvé TR | Soumettre template TR à Meta |
| Changement langue pas persisté | AsyncStorage pas écrit | Check `cacheUserLanguage` |
| Variable `{name}` affichée brute | Pas d'interpolation | Passer `{ name }` en 2e param |
| Traducteurs confondent `{0}` et `{count}` | Inconsistance syntaxe | Standardiser sur `{name}` |

### 25.2 — Debug commands

```bash
# Valider cohérence fichiers
python scripts/validate_i18n.py

# Lister clés manquantes dans une langue
python scripts/missing_keys.py --lang tr

# Extraire clés utilisées dans le code
python scripts/extract_i18n_keys.py --source mobile/app/

# Test détection langue
curl http://localhost:8000/v1/i18n/test?text=Merhaba

# Changer langue pour un user
curl -X PATCH http://localhost:8000/v1/settings \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"locale": "tr-TR"}'
```
